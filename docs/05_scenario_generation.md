# Scenario Generation

## Overview

The scenario generator is the core of CausalStress. Given a shock event family (e.g., "market crash"), it produces 200 Monte Carlo trajectories of 25 financial variables over a 60-day horizon. Each trajectory is a causally-grounded path conditioned on stressed-market dynamics, generated by a regime-conditioned VAR model with causal shock propagation and Student-t innovations.

The canonical model was locked on 2026-04-19 and all paper results derive from it.

**Headline performance (7 out-of-sample test events):** 90.0% coverage, 77.6% direction accuracy, 100% pairwise consistency.

## Step-by-Step Algorithm

### Step 1: Fit the Regime-Conditional VAR

A Vector Autoregression of order 2 (VAR(2)) is fit on the 25 core variables, using only data from stressed training regimes (elevated, stressed, crisis days — approximately 3,374 observations).

```python
DEFAULT_TRAINING_REGIMES = ["elevated", "stressed", "high_stress", "crisis"]
```

The VAR(2) model produces:
- Coefficient matrices A_1 and A_2 (each 25×25) — one-lag and two-lag effects
- Residual covariance matrix Σ (25×25) — the innovation covariance

Using stressed-regime data only means the VAR captures how variables co-move *under pressure*: tighter equity-credit co-movement, stronger volatility feedback, faster spread transmission.

### Step 2: Load the Stressed-Regime Causal Graph

```python
canonical_graph = load_canonical_graph(base_dir)
# Returns: {variable: {variable: weight}} for the stressed regime
# 330 edges total
```

This graph is used in Step 4 to propagate the shock to downstream variables.

### Step 3: Initialize Multi-Root Shock

Each event family has a deterministic shock template specifying which variables are initially shocked and at what magnitude (in σ units of their empirical distribution):

| Event Family | Root Shocks |
|-------------|-------------|
| `market_crash` | ^GSPC −3.0σ, ^VIX +3.5σ, XLF −3.5σ, BAMLH0A0HYM2 +3.0σ, DGS10 −1.5σ |
| `sovereign_crisis` | ^GSPC −3.0σ, ^VIX +3.0σ, BAMLH0A0HYM2 +2.5σ, DGS10 −2.0σ, XLF −2.5σ |
| `global_shock` | ^GSPC −2.0σ, ^VIX +2.5σ, CL=F −3.0σ, XLF −1.5σ, BAMLH0A0HYM2 +1.5σ |
| `volatility_shock` | ^GSPC −2.0σ, ^VIX +4.0σ, XLF −2.0σ, BAMLH0A0HYM2 +1.0σ |
| `rate_shock` | ^GSPC −1.5σ, ^VIX +2.0σ, DGS10 +3.0σ, TLT −2.5σ, XLF −1.5σ |
| `credit_crisis` | ^GSPC −3.0σ, ^VIX +3.0σ, XLF −3.0σ, BAMLH0A0HYM2 +3.0σ, DGS10 −1.0σ |
| `pandemic_exogenous` | ^GSPC −4.0σ, ^VIX +5.0σ, CL=F −5.0σ, XLF −4.0σ, BAMLH0A0HYM2 +4.0σ, DGS10 −2.5σ |

The eighth family (`market_crash` appears in the multi-root template set as the eighth; `pandemic` and `pandemic_exogenous` cover the pandemic family variants).

**Multi-shock distribution:** Scenarios are generated at five shock severity levels to create fat-tailed output:

| Severity | % of scenarios |
|----------|---------------|
| 3.0σ | 40% |
| 4.0σ | 25% |
| 5.0σ | 20% |
| 6.0σ | 10% |
| 7.0σ | 5% |

The template above shows the 3σ anchors; 4σ and above scale proportionally.

### Step 4: Causal Propagation

After the root variables are shocked, the shock propagates through the causal graph:

```
For each directly shocked variable v:
    For each descendant w at hop depth d (d = 1, 2, 3):
        propagated_shock[w] += initial_shock[v] × decay^d
        where decay = 0.4
        capped to [0.12, 2.5]
```

- **Propagation depth:** 3 hops maximum.
- **Decay factor:** 0.4 per hop. A 3σ shock transmits as: 3σ → 1.2σ (hop 1) → 0.48σ (hop 2) → 0.19σ (hop 3), before capping.
- **Min/max bounds:** Propagated shocks below 0.12 are suppressed; above 2.5 are capped. This prevents numerical explosion through highly connected subgraphs.

The causal graph ensures shock direction is economically coherent: if equities fall, the graph propagates a positive shock to ^VIX, positive to HY spreads, negative to bank sector.

**VIX special handling:**
- ^VIX initial shock is capped at ±3.5σ (VIX_TEMPLATE_CAP = 3.5) regardless of family template.
- ^VIX propagation scale = 0.65 (subdued relative to other variables — avoids unrealistic VIX spikes).

### Step 5: Shock Persistence

Financial crises don't resolve in one day. Shocks persist for multiple days with exponential decay:

```
shock_day_t = initial_shock × decay_rate^t
where decay_rate = 0.72, over persistence_days = 5
```

On day 1: full shock. Day 2: 72% of original. Day 3: 52%. Day 4: 37%. Day 5: 27%. From day 6 onward, the initial shock has decayed below the noise threshold and the VAR dynamics take over.

### Step 6: Student-t Innovations

At each of the 60 simulation steps, the VAR model computes the predicted next-day values plus an innovation (shock) term drawn from a multivariate distribution. The canonical model uses Student-t innovations rather than Gaussian:

| Regime context | Degrees of freedom | Calibration source |
|----------------|-------------------|--------------------|
| Normal days (df_normal) | **5.97** | MLE on calm+normal VAR residuals, 2005-2019 |
| Crisis days (df_crisis) | **3.84** | MLE on stressed+crisis VAR residuals, 2005-2019 |
| Mid-regime (df_mid) | **4.79** | Geometric mean of 5.97 and 3.84 |

**Calibration method:** `scipy.stats.t.fit()` applied to pre-2020 VAR residuals, per regime, per variable. The median df across all 25 variables is used. See [ml_pipeline/calibrate_df_from_residuals.py](ml_pipeline/calibrate_df_from_residuals.py).

**Why Student-t beats Gaussian here:** The Kolmogorov-Smirnov test on pre-2020 VAR residuals prefers Student-t over Gaussian for **20/20 core variables in both regime sets**. Financial returns are fat-tailed; Gaussian innovations underestimate the probability of large daily moves and therefore underestimate crisis severity.

**Noise scales:**
- Extreme regime (5σ+ shock scenarios): innovations scaled by 1.2× (CANONICAL_EXTREME_NOISE_SCALE = 1.2)
- Mid-stress regime: innovations scaled by 1.1× (CANONICAL_MID_NOISE_SCALE = 1.1)

**Daily clip:** Each daily move is clipped at ±6σ (DAILY_CLIP_SIGMA = 6.0). This matches the empirical maximum for real crisis events — COVID saw 5.9σ moves in S&P, GFC saw 5.8σ.

### Step 7: Simulate 400 Candidate Trajectories

Run the VAR model forward for 60 days, 400 times (CANONICAL_CANDIDATE_MULTIPLIER = 2 × target 200), with:
- Different random seeds per trajectory
- Student-t innovations at the calibrated df for the regime
- Causal propagation applied at day 1
- Shock persistence applied over days 1-5
- VAR dynamics governing days 6-60

Output: 400 trajectories, each a 60×25 matrix of variable values.

### Step 8: Score and Soft-Filter

Each trajectory is scored for plausibility — a measure of how coherent the trajectory is with the stressed-regime VAR's statistical properties.

```python
# Plausibility score = composite of:
# - How well the trajectory's pairwise correlations match the VAR's covariance matrix
# - Whether the trajectory stays within [p2.5, p97.5] of historical stressed-regime returns
# - Whether directional relationships (equity down → VIX up) hold throughout

plausibility_score = score_canonical_plausibility(trajectory, var_model, causal_adj)
```

**Soft filtering:** Rather than discarding low-plausibility trajectories (hard top-k), the canonical model uses weighted sampling:

```python
weights = score ** 6.0          # steep power law — strongly suppresses low scores
weights = max(weights, 1e-6)    # minimum floor to preserve diversity
# Sample 200 from 400 using these weights
```

The exponent 6.0 was chosen to strongly penalize implausible trajectories (score < 0.5 gets weight < 0.016 of a perfect-scoring trajectory) while preserving some tail diversity. Hard top-k with threshold 0.5 would discard those trajectories entirely.

**Output:** 200 weighted scenarios. Final plausibility distribution (canonical test events): mean = 0.7706, min = 0.449, max = 0.953.

## Event-Specific Guardrails

Some event families have additional per-variable guardrails to prevent unphysical trajectories:

**market_crash:**
- ^VIX: cumulative bounds [50%, 450%], daily abs cap 18%
- CL=F: cumulative abs cap 45%, daily abs cap 8%
- DGS10: cumulative abs cap 120bps, daily abs cap 3bps

**credit_crisis:**
- ^VIX: cumulative bounds [40%, 400%], daily abs cap 16%
- BAMLH0A0HYM2: must be positive (spreads can't tighten in a credit crisis), daily abs cap 20bps

**rate_shock:**
- DGS10: must be positive (rates rising), daily abs cap 5bps

## Evaluation Metrics

**Coverage:** A variable is "covered" if the actual cumulative outcome over the event window falls within the 5th–95th percentile of the 200 generated scenarios. Event coverage = fraction of the 6 key variables covered. Method coverage = mean of event coverages, averaged over 5 random seeds.

**Direction:** A variable's direction is "correct" if the median scenario trajectory has the same sign as the actual outcome (e.g., if the S&P fell, does the median scenario also fall?).

**Pairwise:** Checks directional constraints between variable pairs for the event type (e.g., in a credit crisis: ^GSPC must be down, ^VIX must be up, BAMLH0A0HYM2 must be up). 100% means all pairwise constraints are satisfied on the median trajectory.

**Plausibility:** Mean plausibility score of the 200 output scenarios. Calibrated against the stressed-regime VAR's statistical properties.

## Research Branches (Not in Canonical Path)

### VECM Engine
[ml_pipeline/generative_engine/vecm_engine.py](ml_pipeline/generative_engine/vecm_engine.py) implements a Vector Error Correction Model for cointegrated variable groups. The Johansen test identifies 6 cointegrating vectors across 4 variable groups (yield curve, credit spreads, equity-volatility, macro fundamentals). VECM captures long-run equilibrium pull-back that VAR destroys by differencing — relevant for scenarios where the system needs to mean-revert to an economic equilibrium.

VECM is not yet in the canonical production path. Evidence from Experiment 6 supports its inclusion as a future extension.

### Copula Engine
[ml_pipeline/generative_engine/copula_engine.py](ml_pipeline/generative_engine/copula_engine.py) implements Student-t copula fitting for 12 core variables. The joint copula df = 2.50, giving tail dependence of 0.186 (vs 0.0 for Gaussian). Stressed-regime copula shows 48% higher average correlation than calm-regime (0.171 vs 0.115). This confirms that tail co-movement increases during crises — the "everything crashes together" effect. The copula is not yet integrated into the canonical generator.
